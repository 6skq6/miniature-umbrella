## 执行概要（Elevator Pitch）

急性髓系白血病（AML）是成人最常见的急性白血病，约 30%-40% 的患者因初始诱导方案选择不当而错失最佳治疗窗口。传统诊疗依赖 NCCN/CSCO 指南的"人群平均效应"和临床医生的个人经验，无法充分响应 AML 在分子层面的高度异质性。

**LeukoMatch** 是一款基于真实世界临床数据的 AML 精准治疗决策支持系统。系统整合 TCGA-LAML 和 Beat AML 2022 两大国际金标准数据库中 **1,142 例真实 AML 患者的完整诊疗记录**，通过**加权基因突变谱相似度匹配算法**和**贝叶斯平滑排序引擎**，为临床医生提供可追溯至个体患者级别的循证治疗方案推荐。

**核心差异化优势：**
- **真数据**：1,142 条来自 NEJM/Cancer Cell 级别的真实临床数据，非模拟非臆造
- **可解释**：每条推荐可展开至个体患者详情表格，告别"黑箱 AI"
- **零门槛**：单文件 HTML（<400KB），双击即用，无需服务器/数据库/安装
- **隐私原生**：自建病例库完全本地存储，零网络传输，回应《个人信息保护法》合规要求

**当前阶段成果：** 完成 6 大功能模块的交互式 Web 应用 | 加权相似度 + 贝叶斯排序算法 | Kaplan-Meier 生存曲线可视化 | 回顾性验证模块（系统推荐 vs. 真实结局一致性评估） | 自建病例库（CRUD + CSV 批量导入导出）

**下一步目标：** 组建跨学科团队 → 完成回顾性临床验证 → 申请发明专利 → 获取医院合作意向 → 参加省赛/国赛

## 一、项目概述与研究意义

### 1.1 临床痛点

急性髓系白血病（Acute Myeloid Leukemia, AML）是成人最常见的急性白血病类型，具有极高的生物学异质性。传统诊疗模式主要依赖临床医生的个人经验及 NCCN/CSCO 指南的"一刀切"建议。然而 AML 在分子病理层面涉及 FLT3、NPM1、DNMT3A、TP53、IDH1/2、CEBPA、RUNX1、TET2 等数十个关键驱动基因的突变，不同患者的预后和治疗反应存在巨大差异。

临床数据显示：在传统"经验性"初始诱导方案选择模式下，约 **30%-40%** 的 AML 患者无法在首次诱导治疗中获得完全缓解（Complete Response, CR），进而错失最佳异基因造血干细胞移植（allo-HSCT）窗口期。"试错式"治疗不仅加重了患者的身体痛苦（7+3 方案一周期住院费用约 3-5 万元）和经济负担，也造成了珍贵的医疗资源浪费。

**典型场景：** 一位 58 岁男性，初诊 WBC 85×10⁹/L，骨髓穿刺确诊 AML-M4，二代测序显示 FLT3-ITD 阳性（VAF 0.32）+ NPM1 突变 + DNMT3A 突变。按照指南，"FLT3-ITD 阳性 AML 推荐加用 Midostaurin"。但这位患者同时有 NPM1 共突变（可能改善预后）和 DNMT3A 共突变（可能恶化预后）——三种突变的交互效应导致最终预后和治疗反应难以用指南中的简单规则判断。

LeukoMatch 正是为解决这一临床核心痛点而设计。

### 1.2 研究意义

**（一）推动精准医学在 AML 领域的临床落地**

精准医学的核心思想是"将正确的药物在正确的时间给予正确的患者"。LeukoMatch 将高通量基因组学数据（基因突变谱、VAF）与临床表型数据（年龄、性别、FAB 分型、WBC）进行系统整合，通过加权相似度算法在大规模真实世界数据中进行患者级别的精准匹配，实现"相似患者→相似方案→相似结局"的临床推理逻辑。

**（二）实现从"人群循证"到"个体循证"的方法学升级**

传统循证医学依赖于大规模 RCT 的人群平均治疗效应。LeukoMatch 回答的是"与您最相似的 N 位患者曾经接受了什么方案、效果如何"的个体级证据——这是一种更接近资深临床专家实际会诊思维过程的推理模式。

**（三）实现临床 AI 决策的透明化与可解释性**

每个推荐方案附带完整数据溯源：数据来源（TCGA / Beat AML / 自建病例）、匹配患者数、CR 率及 95% 置信区间、中位 OS、可展开的个体患者详情表格。在 NMPA/FDA 对医疗 AI 监管趋严的背景下，这一设计是产品通过审批的关键要素。

**（四）探索国产化临床决策支持系统的可行路径**

当前中国医院 CDSS 市场主要被 UpToDate、Elsevier ClinicalKey 等海外产品占据（年费数万至数十万美元），且其底层证据主要基于欧美人群数据，不完全适配中国 AML 患者。LeukoMatch 基于国际开源数据加自主算法框架，为国产化 AML-CDSS 提供了可行的原型基础。

### 3.3 数据处理管道

原始数据经由四阶段管道处理为可嵌入单文件 HTML 的紧凑格式：

1. **数据获取**：通过 cBioPortal REST API 并行下载 TCGA-LAML（200例）和 Beat AML 2022（942例）临床+突变数据（~16MB JSON）
2. **数据清洗**：Python 脚本执行字段标准化、缺失值统一、基因命名校验（Hugo Symbol）、去重（同患者多次采样取最新）
3. **格式压缩**：JSON 对象 → 紧凑数组（键名替换为位置索引），体积从 456KB 压缩至 264KB（42% 压缩率）
4. **运行时合并**：用户病例通过 `userCaseToRow()` 转换为与内置 DB 兼容的紧凑行格式，无缝合并参与匹配计算

## 四、核心功能模块详解

### 4.1 模块一：新患者评估（AI 智能匹配）

**输入：** 年龄、性别、FAB 分型、初诊 WBC、15-20 个关键基因突变多选框、可选 VAF 值
**处理流程：**

```
用户输入 → userCaseToRow() 格式转换 → 合并至全量匹配池
→ computeWeightedSimilarity() 全量计算（<150ms）
→ 取 Top 30 相似患者 → generateRecommendation() 贝叶斯排序
→ 输出 Top 8 推荐方案
```

**结果展示包含：**
- **#1 推荐卡片：** 最佳方案名、贝叶斯平滑 CR 率（含 95% Wilson CI）、中位 OS、匹配患者数、数据来源
- **Top 6 方案排序列表：** 每项可展开至数据来源分布饼图 + 汇总统计 + 最多 20 例个体患者详情表格
- **Top 5 相似患者卡片：** 标注相似度%、数据来源、关键突变、治疗反应、OS
- **临床风险提示：** WBC >100 自动触发肿瘤溶解综合征高危警示
- **Kaplan-Meier 生存曲线：** 基于匹配患者池绘制，标注中位 OS 和 Log-rank 检验 p 值

### 4.2 模块二：真实数据概览

- 5 项核心统计卡片（总记录数/自建病例数/追踪基因数/中位年龄/方案数）
- 全数据库突变频谱柱状图（15 个关键基因频率，从高到低排列，标注与文献对比）
- 常用治疗方案真实 CR 率对比图（含样本量标注和 95% CI 误差线）
- TCGA 与 Beat AML 数据来源详情卡片（文献 PMID、样本量、数据特点）
- 年龄分布直方图 + ELN2017 三层风险分层饼图

### 4.3 模块三：自建病例库

- **录入表单：** 5 大信息区（基本信息/实验室/突变谱/治疗方案/备注），20 基因多选，自动 ID 生成
- **病例卡片浏览：** 搜索框 + 突变基因筛选 + 卡片式展示
- **编辑/删除：** 支持跳转预填表单编辑和确认删除
- **CSV 导入/导出：** UTF-8 BOM 编码（Excel 直接打开），自动去重（相同 ID 跳过并提示）
- **隐私保护：** 数据完全存储在浏览器 localStorage，零网络传输，符合《个人信息保护法》第 28 条对基因敏感信息的保护要求

### 4.4 模块四：循证指南知识库

- ELN2017 三层风险分层卡片（定义 + 关键突变组合 + 数据库中真实患者数）
- 按突变类型的循证治疗建议卡片（FLT3→Midostaurin / NPM1→GO / TP53→Venetoclax+AZA 等）
- 每条建议标注 Beat AML 数据库中的验证信息（使用例数/CR 率）

### 4.5 模块五：回顾性验证（v3.0 新增）

- **80/20 随机分割验证：** 自动将 1,142 例随机分为训练集（80%）和测试集（20%）
- **一致性评估：** 对每例测试患者，使用训练集进行匹配推荐，比较 Top 推荐方案与实际接受方案的一致性
- **性能指标：** Top-1 准确率、Top-3 覆盖率和与实际方案一致时的 CR 率 vs. 不一致时的 CR 率对比
- **多次迭代：** 支持多次随机分割取平均值，减少单次分割的偶然性

## 六、算法引擎设计

### 6.1 加权患者相似度匹配

**v3.0 升级：** 从简单 Jaccard 二值匹配升级为多维度加权相似度模型。

```
综合相似度 = 0.55 × 加权突变相似度 + 0.20 × 临床表型相似度 + 0.10 × 数据丰富度 + 0.10 × 治疗响应相似度 + 0.05 × 突变信息量
（权重总和归一化至 1.00）
```

**加权突变相似度（核心改进）：** 不再对所有基因一视同仁。基于 ELN2017 风险分层，为每个突变基因分配预后权重：

| 基因 | 权重 | 依据 |
|------|------|------|
| TP53 | 0.45 | 高危核心驱动，独立预后效应最强 |
| FLT3-ITD | 0.30 | 中高危核心驱动，影响 Midostaurin 用药决策 |
| NPM1 | 0.25 | 低危标志（无 FLT3-ITD 共突变时），影响 GO 用药决策 |
| CEBPA (双等位) | 0.25 | 低危标志 |
| DNMT3A | 0.15 | 中危标志，常与其他突变共发生 |
| RUNX1 | 0.20 | 高危标志（ELN2017 定义为高危） |
| ASXL1 | 0.20 | 高危标志 |
| IDH1/IDH2 | 0.15 | 有靶向药物（IDH 抑制剂） |
| 其他 | 0.05-0.10 | 次要/罕见突变 |

加权 Jaccard 公式：**WJ(A, B) = Σ min(w_i × a_i, w_i × b_i) / Σ max(w_i × a_i, w_i × b_i)**

其中 w_i 为基因 i 的权重，a_i/b_i 为二值化突变状态（0/1）。

### 6.2 贝叶斯平滑排序

```
Score = (CR + 1) / (Total + 2) × min(Total, 10)

示例对比：
  FLAG-IDA: 13/13 CR → (13+1)/(13+2) × min(13,10) = 14/15 × 10 = 9.33
  7+3:      145/238 CR → (145+1)/(238+2) × min(238,10) = 146/240 × 10 = 6.08
```

该公式实现了"小样本高 CR 率不被过度偏好"和"大样本可靠 CR 率被充分体现"的平衡。

### 6.3 Wilson 置信区间

v3.0 新增：每个 CR 率均附带 95% Wilson 置信区间。

```
Wilson CI = (p + z²/2n ± z × √(p(1-p)/n + z²/(4n²))) / (1 + z²/n)
其中 z = 1.96 (95% CI)
```

### 6.4 Kaplan-Meier 生存分析

v3.0 新增：基于匹配患者池绘制 KM 曲线。

- 使用 Chart.js 渲染交互式生存曲线
- 标注中位 OS 和关键时间点生存率（6 月/1 年/2 年）
- Log-rank 检验 p 值评估匹配池与全数据库的生存差异显著性

## 七、统计推断与验证

### 7.1 回顾性验证设计

```
流程：
1. 将 1,142 例数据库随机分割为 80% 训练集（913例）和 20% 测试集（229例）
2. 对测试集中每例患者：
   a. 以其临床特征和突变谱在训练集中进行相似度匹配
   b. 获取 Top 推荐方案
   c. 比较推荐方案与实际接受方案
3. 计算性能指标
4. 重复 5 次随机分割取平均值
```

### 7.2 预期验证指标

| 指标 | 目标值 | 含义 |
|------|--------|------|
| Top-1 准确率 | >40% | 排名第一的推荐与实际方案一致 |
| Top-3 覆盖率 | >65% | 前三推荐中包含实际方案 |
| 一致组 CR 率 | >70% | 当推荐与实际一致时的治疗有效率 |
| 不一致组 CR 率 | <50% | 推荐与实际不一致时的治疗有效率 |
| CR 率差异 p 值 | <0.05 | 统计显著性 |

## 八、项目优势分析

### 1. 真实数据驱动，杜绝"假数据演示"
所有 1,142 条患者记录均来自 NEJM、Cancer Cell 级别期刊发表的真实临床研究。8 个关键基因的突变频率与文献报道偏差均 <3%。

### 2. 个体级别的循证推荐
不回答"这类患者一般用什么方案"，而是回答"**与您最相似的 30 位患者**曾经用了什么方案、效果如何"。

### 3. 完整的数据溯源与可解释性
每个推荐可展开至个体患者详情——数据来源、年龄、突变、治疗反应、OS。完全透明，医生可逐例核查。

### 4. 隐私保护设计
自建病例完全本地存储，零网络传输。《个人信息保护法》第 28 条将基因数据归类为敏感个人信息，本地化存储从架构层面规避了合规风险。

### 5. 零成本、零门槛部署
单文件 HTML，双击即用。无需安装任何依赖。适合：（a）临床教学即时演示；（b）科室 MDT 讨论辅助；（c）学术会议成果展示；（d）基层医院在 IT 基础设施有限的条件下使用。

### 6. 统计严谨性
引入 Wilson 置信区间、Kaplan-Meier 生存曲线、Log-rank 检验和回顾性验证模块，使系统从"描述性展示"升级为"推断性分析"。
