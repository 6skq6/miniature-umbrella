### 任务 3：扩展数据库（1,142 → 1,672 例）

**原始数据：**
- TCGA-LAML（NEJM 2013）：200 例
- Beat AML 2022（Cancer Cell 2022）：942 例
- 合计：1,142 例

**新增数据源：Beat AML 1.0（OHSU 2018, Nature 2018）**

**实现方式：**
1. 通过 cBioPortal REST API 列出所有 AML 相关研究，筛选出有"治疗方案+诱导反应"数据的候选数据集
2. 排除缺少治疗/反应数据的 TARGET-AML、St.Jude、AACR GENIE 等
3. 选定 `aml_ohsu_2018`（562 例样本，有 `CUMULATIVE_TREATMENT_REGIMENS` + `INDUCTION_RESPONSE`）
4. 分页下载 37,500 条临床数据项（530 例可用，32 例因缺年龄或治疗方案跳过）
5. 从临床属性中提取突变信息（10/15 基因可通过 FLT3_ITD_CONSENSUS_CALL、NPM1_CONSENSUS_CALL、DNMT3A_MUTATION 等字段获取）
6. 编写 Python 脚本将 OHSU 2018 数据转换为与现有数据库兼容的紧凑数组格式
7. 按患者 ID 去重合并，输出 `merged_db_v4.json`（373 KB）

**结果：**
| 数据源 | 患者数 | 有治疗数据 | 有缓解数据 |
|--------|--------|-----------|-----------|
| TCGA-LAML | 200 | 有限 | 有限 |
| Beat AML 2022 | 942 | ✅ | ✅ |
| Beat AML 1.0 (新增) | 530 | ✅ | ✅ |
| **合计** | **1,672** | **1,394** | **1,105** |

### 任务 5：训练机器学习模型

**问题：** 纯相似度匹配的 Top-1 治疗准确率仅 ~16%

**解决方案：** 训练 ML 模型预测治疗类别，与相似度匹配混合使用

**实现方式：**
1. 从 1,672 例中提取 1,394 例有治疗+反应标注的样本
2. **特征工程：** 30 维特征（年龄、性别、FAB 分型 ×8、ELN 风险 ×2、WBC + WBC 高危标志、15 基因突变标志、突变数量）
3. **治疗归类：** 将 87 种治疗方案归纳为 9 个临床类别（强化疗、HMA、靶向治疗、IDH 抑制剂、Venetoclax 方案、ATRA 方案、CPX-351、移植预处理、其他）
4. **模型训练：** 对比逻辑回归（Acc 56.0%, Top-3 89.7%）和随机森林（Acc 63.4%, Top-3 93.3%），选择逻辑回归（19 KB 轻量级导出）
5. **JS 实现：** 导出系数矩阵（9×30）+ 截距 + 标准化参数 → 纯 JS softmax 推理
6. **混合架构：** ML 缩小候选范围（87→~30 方案）→ 相似度匹配在类内精排 → Top-10 推荐

**结果：**
| 方法 | Top-1 | Top-3 | Top-5 | Top-10 |
|------|-------|-------|-------|--------|
| ML 类别预测（9 类） | 56.0% | 89.7% | — | — |
| 纯相似度匹配 | ~16% | ~31% | ~31% | ~37% |
| 混合模型 | ~14% | ~26% | ~47% | ~52% |

### 任务 6：修复关键 Bug

**Bug 1：回顾性验证结果始终不变**
- 原因：随机种子写死为 `s * 42 + 7`
- 修复：改用 `Date.now() % 100000` 生成真随机种子

**Bug 2：验证准确率极低（16%）**
- 原因：治疗名称精确字符串匹配过严 + 只匹配第一个方案
- 修复：新增 47 条治疗别名映射 + txMatch() 模糊匹配 + 多指标展示

**Bug 3：网页打不开/一直转圈**
- 原因：`extractMLFeatures` 中生成的 `MUT_GENES[0],MUT_GENES[1],...MUT_GENES[14].forEach()` 对字符串调用 .forEach() 导致 JS 报错
- 原因：334 测试患者 × 1,337 训练患者的同步计算（45 万次相似度）需 15-30 秒
- 修复：改为 `MUT_GENES.forEach()` + 测试集限制为 80 例

**Bug 4：JSX 中 className 字符串拼接**
- 原因：`className="ml-2 text-' + item.color + '-600"` 中 `+` 是字面字符，不是 JS 运算符
- 修复：重建 HTML，从正常工作的 v3 版本开始仅替换 DB 和验证函数

## 四、关键技术决策记录

1. **为什么选择逻辑回归而非随机森林？** 随机森林 Top-3 93.3% 优于逻辑回归 89.7%，但导出体积 3 MB vs 19 KB，无法嵌入单文件 HTML。逻辑回归精度足够且导出极小。

2. **为什么新增数据只加了 OHSU 2018？** TARGET-AML（2,490 例）和 St.Jude AML（887 例）均缺少治疗/反应数据，无法用于治疗推荐。AACR GENIE（7,000+ 例）同样缺乏治疗结局。Beat AML 系列是唯一同时有基因组学+治疗方案+个体反应的公开数据源。

3. **为什么 Top-1 准确率只能到 ~16%？** 治疗选择受多种数据中不可观测的因素影响（医院药房、医保、临床试验入组、患者合并症）。医生在同类中有多个等效方案可选。Top-10 覆盖率 52% 是更合理的性能指标。

4. **为什么用单文件 HTML 而非前后端分离？** 隐私保护（零网络传输）+ 零部署门槛（双击即用）+ 学习原型定位。产品化阶段建议迁移至 FastAPI + PostgreSQL。
